# プロジェクト: Pathieve (パシーヴ)

## 1. プロダクトコンセプト
- **コンセプト:** 未来への道筋をいつでも描き直せる、動的な目標管理ツール。
- **キャッチコピー:** 「あなたの目標達成前夜」
- **ネーミング由来:** Path（道） + Achieve（達成） + Eve（前夜）。目標達成に向けた準備期間や、試行錯誤する過程を支えることを重視する。

## 2. 開発における重要指針（AIへの要求）
- **柔軟な設計:** 「道筋をいつでも描き直せる」というコンセプトに基づき、データの変更や履歴管理が容易な構成を優先すること。
- **ユーザー体験:** ユーザーが目標に向かって前向きに「準備」を楽しめるよう、エラーメッセージやUIのテキストは冷徹なものではなく、建設的でサポートを感じるトーンを意識する。
- **UI/UXの提案:** 複雑な進捗管理よりも、直感的に「道筋（Path）」を操作できるインターフェースを提案すること。

## 3. 技術スタックの定義
本プロジェクトは、主に以下の技術スタックで構成されています。（※PHP/MVCではなく、モダンなTypeScriptエコシステムを採用しています）
- **フレームワーク:** Next.js (App Router) / React 19
- **言語:** TypeScript
- **データベース & ORM:** MySQL + Prisma ORM
- **スタイリング:** Tailwind CSS v4 + Vanilla CSS (`globals.css`のCSS変数によるカスタムテーマ)
- **認証:** `bcryptjs` と `jose` を用いたカスタム処理 (Next.js Route Handlers上)

## 4. コーディング規約
- **命名規則:**
  - データベース/DBスキーマ (Prisma): テーブル名やカラム名には `snake_case` を使用 (`users`, `created_at` 等)。
  - TypeScript (変数, 関数, メソッド): `camelCase` を使用。
  - React コンポーネント・ファイル・ディレクトリ: コンポーネントは `PascalCase`、ディレクトリはNext.jsの規約に沿うか `kebab-case`。
- **型定義と構文:** TypeScriptの厳格(strict)な型チェックを順守する。安易な `any` は避け、`types/` ディレクトリ等で適宜型・インターフェースを定義する。
- **コード品質:** ESLintのルール (`eslint.config.mjs`) に準拠した美しいコードを保つ。

## 5. 実装上の制約・アーキテクチャ
- **ビジネス・APIロジック:** 
  Next.js の Route Handlers (`app/api/**/route.ts`) にビジネスロジックを実装する。
- **データベース操作:** 
  `lib/prisma.ts` の Prisma Client を用いる。生SQLは極力避け、Prismaの機能を利用して操作する。
- **フロントエンドのAPI通信:**
  APIの直接呼び出し（fetch等）は各コンポーネントに散財させず、`lib/api.ts` などのクライアント層を介して行う。
- **状態管理・UI:**
  全体に関する状態管理は React Context (`context/` ディレクトリ) を活用。UIコンポーネントは再利用性を考慮し `components/` ディレクトリに配置する。
